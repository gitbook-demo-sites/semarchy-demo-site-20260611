# Releases

SaaS should keep a simple release track. Readers need to understand what changed recently, what is coming, and whether any behavior changed in ways that affect configuration or integrations.

## Suggested release content

- recent product updates
- deprecations
- rollout notes
- compatibility callouts when a tool or API workflow changes
