# Design CLI

The Design CLI belongs in a shared builder space because it is neither purely SaaS nor purely self-hosted. Its guidance should cover:

- install and bootstrap
- local design workflow
- model validation
- CI or automation usage
- compatibility expectations with platform tracks

{% hint style="info" %}
The demo point here is structural: once a reader is on a Design CLI page, they should be one click away from platform-track context and one click away from the compatibility matrix.
{% endhint %}
